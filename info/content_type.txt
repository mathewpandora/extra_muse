Content-Type — это инструкция серверу: «в каком формате я отправляю тебе данные в теле запроса».

Когда нужно указывать (всегда, если есть тело запроса)

Для методов POST, PUT, PATCH — когда вы передаёте какие-то данные в теле (JSON, форму, файл, XML и т.д.).

application/json	Отправляете данные в формате JSON (почти все современные REST API)
application/x-www-form-urlencoded	Отправляете пары ключ=значение, как обычная веб-форма (пример: name=John&age=30)
multipart/form-data	Отправляете файлы или смесь полей с файлами (загрузка фото, документов)
text/plain	Отправляете обычный текст без форматирования (редко в API, чаще для отладки)
application/xml	Отправляете XML (старые SOAP-сервисы)
